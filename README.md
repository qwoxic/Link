# Техническое задание проекта «WorkFlow»
**Тема:** Веб-мессенджер с интегрированным Канбан-трекером для командной разработки.

## 1. Основные функциональные требования

### 1) Система авторизации и профилей
*   **Регистрация и вход:** Полноценная система аутентификации (Flask-Login / Django Auth).
*   **Безопасность:** Обязательное хэширование паролей пользователей.
*   **Профили:** Личный кабинет пользователя с возможностью загрузки аватара и редактирования личных данных.

### 2) Мессенджер и коммуникация
*   **Групповые чаты:** Возможность создавать проекты, которые автоматически становятся групповыми чатами для участников.
*   **Обмен сообщениями:** Отправка текстовых сообщений с сохранением истории в БД.
*   **Файловое хранилище:** Поддержка загрузки файлов (документов, логов, изображений) прямо в чат проекта.

### 3) Интеграция с Task-трекером (Канбан)
*   **Трансформация данных:** Возможность одним кликом превратить сообщение из чата в задачу (Task) для проекта.
*   **Управление задачами:** Создание, редактирование и назначение задач на конкретных участников проекта.
*   **Канбан-доска:** Визуальное отображение задач в трех колонках: «В планах», «В работе», «Готово».

### 4) API и дополнительные функции
*   **REST API:** Реализация эндпоинта для получения списка активных задач проекта в формате JSON.
*   **Поиск:** Глобальный поиск по сообщениям внутри чата и по заголовкам задач.

---

## 2. Архитектура и дизайн

### Основные модули
1.  **Auth Module:** Управление доступом, регистрация и сессии пользователей.
2.  **Chat Engine:** Обработка логики сообщений и связей между пользователями в проекте.
3.  **Task Manager:** Модуль управления состояниями задач и их отображения на Канбан-доске.
4.  **File Storage:** Обработка и валидация загружаемых файлов, сохранение путей в БД.

### Структура интерфейса (Bootstrap 5)
*   **Navigation Bar:** Поиск, уведомления, переход в профиль.
*   **Workspace:** 
    *   Левая панель: Список активных чатов/проектов.
    *   Центральная часть: Переключатель между вкладками «Чат» и «Канбан-доска».
*   **Modals:** Всплывающие окна для создания задач и редактирования профиля.

---

## 3. Схема базы данных (ORM-модели)
*   `User`: (id, username, email, password_hash, avatar_url)
*   `Project`: (id, title, description, owner_id)
*   `ProjectMember`: (id, user_id, project_id) — связь многие-ко-многим.
*   `Message`: (id, project_id, user_id, content, file_path, created_at)
*   `Task`: (id, project_id, title, status, executor_id)

---

## 4. Технологический стек
*   **Backend:** Python, Flask / Django.
*   **Database:** SQLAlchemy (SQLite для разработки, PostgreSQL для продакшена).
*   **Frontend:** HTML5 (Jinja2), CSS3 (Bootstrap 5), JavaScript (Fetch API).
*   **Hosting:** PythonAnywhere.
